# Security Policy

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Reporting a Vulnerability

We take the security of Presidential Punch Peru seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### Please Do NOT:
- Open a public GitHub issue for security vulnerabilities
- Discuss the vulnerability in public forums or social media

### Please DO:
1. **Email us** at the project maintainer's contact (available in GitHub profile)
2. **Include detailed information** about the vulnerability:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested fixes or mitigations
   - Your contact information (optional, for follow-up)

### What to Expect:
- **Acknowledgment**: We'll acknowledge receipt of your report within 48 hours
- **Assessment**: We'll assess the vulnerability and determine its severity
- **Updates**: We'll provide regular updates on our progress
- **Resolution**: We'll work to fix the issue promptly and release a patch
- **Credit**: We'll credit you in our security advisories (if you wish)

### Response Timeline:
- **Critical vulnerabilities**: 24-48 hours
- **High severity**: 7 days
- **Medium/Low severity**: 30 days

## Security Best Practices

### For Contributors:
- Keep dependencies up to date
- Use secure coding practices
- Never commit sensitive information (API keys, passwords, etc.)
- Review code for potential security issues before submitting PRs

### For Users:
- Keep your installation up to date
- Report any suspicious behavior
- Use HTTPS when accessing the application
- Be cautious with any user-generated content

## Data Security

Since this project deals with political information:
- All data should be from public, verifiable sources
- No personal or private information should be included
- User interactions (if any) should not store sensitive data
- All political content should be factual and properly sourced

## Dependency Security

We regularly:
- Monitor dependencies for known vulnerabilities
- Update dependencies to secure versions
- Use automated security scanning tools
- Review new dependencies for security implications

## Contact

For general security questions or concerns, please contact the project maintainers through GitHub issues (for non-sensitive matters) or directly via the contact information in the repository.
